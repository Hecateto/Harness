**Q：精读这篇论文并输出详细的报告。**

不要一上来就想着"输出报告"。你先把论文的核心转折（the turn）找到——作者到底推翻/修正了之前领域里的哪个共识？如果找不到这个转折点，报告就会变成高级摘要，没价值。

**追问**：如果我把报告写成六段式（背景-方法-实验-结论-Q&A-总结），是不是太模板化了？

六段式本身不是问题，但"为了填完六个格子而硬凑内容"是问题。如果你发现某个格子没什么可说的（比如实验设计平淡无奇），直接删掉，不要让格式绑架内容。报告质量的上限由最有洞察力的那两段决定，不由最完整的那段决定。

---

**Q：论文里一堆术语（PoT、PAL、PEV、AHE……），我需要在报告里全部解释吗？**

不是"全部解释"，而是"首次出现时给一句话定位"。读者在看到缩写的那一刻，最需要知道的是"这东西是干嘛的"，而不是它的全称。超过15字的解释说明你自己也没理解到位。

反面教材：在报告末尾挂一个"术语表"。读者读到中间已经卡壳了，谁有耐心翻到结尾查表？

---

**Q：Q&A 部分怎么设计问题才能不流于形式？**

问题必须从你对论文的**具体不满**或**具体好奇**里长出来。模板化的问题套路（"本文的创新点是什么？""未来方向有哪些？"）本质上是在 asking for a summary，不是在 thinking。

好问题的信号：
- 作者说 X 能解决这个问题，但如果条件 Y 不成立，X 会不会反而制造更大的麻烦？
- Table 3 显示 A 比 B 好 5%，但这 5% 来自方法本身还是数据集偏置？
- 如果我把论文里的核心模块替换成一个更简单的版本，假设它还能 work，那说明作者的贡献其实是别的什么东西。

**追问**：那我每次都要问够 5 个问题吗？

看论文质量和你的思考空间。2 个真正扎进去的问题远胜 5 个泛泛而谈。但如果论文确实在多个层面都有张力，5 个也可以。数量随内容复杂度弹性浮动，不设定额。

**Q：作者列了一堆 future work，我是不是都要写进报告？**

区分"礼貌性展望"和"实质性开放问题"。

- 礼貌性展望的标志："未来可以探索更大的模型/更多语言/更多场景"——这话放在任何论文结尾都成立，没有信息增量。
- 实质性开放问题的标志：作者暴露了自己框架的结构性裂缝（"我们的方法在 X 假设下有效，但 X 在真实场景中通常不成立"），或者提出了可操作的子问题（"需要定义新的 evaluation metric 来分离 Y 和 Z"）。

只保留后者。

---

**Q：报告写完之后，我需要关联其他论文形成对比吗？**

不要预设报告之间的关联。每篇报告的使命是穷尽这篇论文自身的 insight 和边界。跨论文的对比应该在阅读多篇之后**自然浮现**，而不是在每篇结尾硬加一个"与相关工作对比"的章节。

如果读完 5 篇之后你发现 3 篇都在攻击同一个隐含假设，那时再单独写一个 cross-paper note，比在每篇里蜻蜓点水有效得多。

**Q：综述类论文（survey）和单篇研究论文的阅读策略应该一样吗？**

绝对不能一样。单篇论文的核心是"作者挖了多深的井"；综述的核心是"这条河从哪流来、有多少支流、哪里还没被探过"。

对于综述：
- 不要拼命记住所有 surveyed work 的名字——那是维基百科的活。
- 要抓住编者的**重新划分**：他为什么把这三类工作归为一组、把那两类拆开到不同章节？这个 grouping 本身就是 argument。
- Open Problems 章节通常比前面所有章节都更有原创性——那是编者真正亮出判断的地方。

**追问**：那综述的 Q&A 怎么设计？

从"编者议程"的角度提问：
- 如果全盘接受这个 taxonomy，我们会忽略哪些被边缘化的研究路径？
- 编者说领域从 A 范式走向 B 范式是必然的，但如果换个角度看，A 的失败是否还有别的解读方式？
- §5 的某个 open problem 如果一旦被解决，会不会反过来推翻 §2 的整个分类基础？

**Q：我觉得报告还是不够"高级"，是不是该加点比喻和类比让内容更生动？**

比喻只在一种情况下有用：当你面对两个容易被混淆的概念，需要用刀切开它们的边界时。比如"PEV Loop 不是让 agent 会调试，而是把控制权从模型手里收回给 harness"——这个类比（控制权回收）是为了区分 PEV 和 ReAct 的本质差异。

装饰性比喻是毒素。"harness 是 agent 的缰绳"这种话听起来漂亮，但如果读者已经懂了 harness 的概念，这句就是噪音；如果没懂，这句也教不会他什么。

---

**Q：最后总结一下，一篇好报告的最核心特征是什么？**

读者关上报告后，脑子里留下的不应该是一串术语和数字，而应该是**一个问题**——一个他自己想继续追问的问题。这个问题可能是对作者假设的质疑，也可能是对领域边界的重新想象。

如果报告读完后读者只想说"这篇论文做了 X，结果是 Y"，那报告失败了。如果读完后他想问"但要是 Z 不成立呢？"，那报告成功了。
